# Template Coverage Matrix

This file is a maintenance contract for Dflow, not the runtime brain. `SKILL.md` should point to this matrix for review and maintenance work instead of duplicating the full table.

The matrix lists Brownfield / Greenfield logical template parity so reviewers can check which templates should remain aligned and which differences are intentional.

## Reference Flow Parity

Common reference flows under `sdd-ddd-brownfield-skill/references/` and
`sdd-ddd-greenfield-skill/references/` must stay synchronized unless a
track-specific difference is explicit. This includes
`dflow-feedback-flow.md`; it is a governance/support flow and should not grow
GitHub CLI submission behavior without a separate proposal.

## Section Anchors

The `Section anchors` column is the single maintenance location for template section anchor coverage. Do not create a separate `SECTION-ANCHORS.md`.

When adding a new anchor:

1. Add the anchor definition to `archive/proposals/PROPOSAL-013-system-document-template-coverage.md` §1 "Important Dflow-updated sections" (historical reference) or its successor governance document.
2. Add the anchor id to the matching row in this matrix.
3. Follow the anchor naming / namespacing / versioning rules defined in `archive/proposals/PROPOSAL-013-system-document-template-coverage.md` §1.
